# Letta SDK 1.7.8 → 1.12.1 Upgrade Findings

**Date:** 2026-07-03  
**Spike:** `spikes/08-letta-sdk-1.12-upgrade-check.ts`  
**Pinned today:** `@letta-ai/letta-client@^1.7.8`  
**Target:** `@letta-ai/letta-client@1.12.1`

## Executive Summary

**Recommendation: safe to upgrade with no production code changes required.**

TypeScript signature comparison across every method this repo calls shows **no breaking changes** between 1.7.8 and 1.12.1 for our usage. Method names, argument order, and request/response shapes for `agents.create/update/delete/list/retrieve`, `passages.create/delete/list/search`, `blocks.retrieve/update`, and `messages.create` are unchanged.

Offline spike verification passed (SDK loads from isolated 1.12.1 install; all 12 required methods present). Live Letta Cloud smoke was **not run** in this environment (`LETTA_API_KEY` unavailable). Before merging Task 12 to production, run the spike once with a real key:

```bash
NODE_PATH=/tmp/letta-sdk-1121/node_modules LETTA_API_KEY=... tsx spikes/08-letta-sdk-1.12-upgrade-check.ts
```

## Spec Assumed vs Verified Reality (1.12.1)

| Area | 1.7.8 (pinned) | 1.12.1 verified | Impact on this repo |
|---|---|---|---|
| Import | `import { Letta } from "@letta-ai/letta-client"` | Same (default + named export both exist) | None |
| Constructor | `new Letta({ baseURL })` / env `LETTA_API_KEY` | Unchanged (`ClientOptions`) | None |
| `agents.create` | `{ name, model, embedding, enable_sleeptime, tools, tags, memory_blocks }` | Same fields; `enable_sleeptime` still optional | None |
| `agents.update` | `{ enable_sleeptime: true }` | Still supported | None |
| `agents.delete` / `list` / `retrieve` | Same signatures | Unchanged | None |
| `passages.create` | `(agentId, { text })` → `Passage[]` | Unchanged | None |
| `passages.delete` | `(passageId, { agent_id })` | Unchanged | None |
| `passages.list` | `(agentId, { limit, after, ascending })` | Unchanged | None |
| `passages.search` | `(agentId, { query, top_k })` → `{ count, results[] }` | Unchanged (`results[].content`, not `.text`) | See pre-existing note below |
| `blocks.retrieve` | `(label, { agent_id })` | Unchanged | None |
| `blocks.update` | `(label, { agent_id, value })` | Unchanged | None |
| `messages.create` | `(agentId, { messages, override_model?, max_steps? })` | Unchanged overloads | None |

## Additive SDK Changes (not used by this repo)

These appear in 1.12.1 types but are **not called** anywhere in `src/` today:

- `agents.recompile()` — new system-prompt recompilation endpoint
- `client.environments` — new top-level resource
- Template rollback/save endpoints
- Extra model-settings union members (SgLang, Moonshot, Baseten, …)
- Message types gained optional `otid` field (idempotency for background streaming)

None require code changes unless we adopt those features later.
